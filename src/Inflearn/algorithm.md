# Inflearn 코딩테스트 문제 요약

기준 자료: `~/file/inflearn`의 섹션 1~7 PDF 및 문제별 풀이 파일.

각 문제는 "무엇을 구하는지"와 "어떤 방식으로 풀면 되는지"만 빠르게 확인할 수 있게 정리했다. 구현할 때는 입력 파싱 후 아래 핵심 아이디어를 그대로 코드로 옮기면 된다.

## 섹션 1. String

| 번호 | 문제 | 간략 설명 | 풀이 핵심 |
| --- | --- | --- | --- |
| 1 | 문자 찾기 | 문자열에서 특정 문자가 몇 번 나오는지 센다. 대소문자는 구분하지 않는다. | 문자열과 목표 문자를 같은 대소문자로 통일한 뒤 한 글자씩 비교해 카운트한다. |
| 2 | 대소문자 변환 | 대문자는 소문자로, 소문자는 대문자로 바꿔 출력한다. | 각 문자의 대소문자 여부를 확인해 반대 케이스로 변환한다. ASCII 값을 이용해도 되고 `Character` API를 써도 된다. |
| 3 | 문장 속 단어 | 공백으로 구분된 문장 안에서 가장 긴 단어를 찾는다. 길이가 같으면 먼저 나온 단어를 선택한다. | `split(" ")` 또는 공백 위치 탐색으로 단어를 순회하며 현재 최대 길이보다 클 때만 답을 갱신한다. |
| 4 | 단어 뒤집기 | N개의 단어를 각각 뒤집어 출력한다. | 각 단어마다 `StringBuilder.reverse()`를 쓰거나 양끝 포인터를 교환한다. |
| 5 | 특정 문자 뒤집기 | 알파벳만 뒤집고 특수문자는 원래 자리에 둔다. | 문자열을 배열로 만들고 양끝 포인터를 둔다. 알파벳이 아니면 해당 포인터만 이동하고, 둘 다 알파벳일 때 교환한다. |
| 6 | 중복문자제거 | 문자열에서 중복 문자를 제거하되 처음 등장한 순서를 유지한다. | 현재 인덱스가 해당 문자의 첫 등장 위치인지 확인하거나, `Set`으로 이미 본 문자를 체크한다. |
| 7 | 회문 문자열 | 공백 없는 문자열이 대소문자 무시 기준으로 회문인지 판별한다. | 소문자 또는 대문자로 통일한 뒤 양끝 문자를 비교한다. 하나라도 다르면 `NO`, 끝까지 통과하면 `YES`. |
| 8 | 유효한 팰린드롬 | 공백과 특수문자를 무시하고 알파벳만으로 팰린드롬인지 판별한다. | 알파벳만 남기고 대소문자를 통일한 뒤 회문 검사한다. 정규식으로 제거하거나 포인터 이동 중 비알파벳을 건너뛴다. |
| 9 | 숫자만 추출 | 문자와 숫자가 섞인 문자열에서 숫자만 뽑아 자연수로 만든다. | 숫자 문자를 만날 때마다 `answer = answer * 10 + digit`으로 누적하면 앞의 0이 자연스럽게 사라진다. |
| 10 | 가장 짧은 문자거리 | 문자열의 각 위치에서 특정 문자까지의 최소 거리를 구한다. | 왼쪽에서 오른쪽으로 목표 문자와의 거리, 오른쪽에서 왼쪽으로 목표 문자와의 거리를 각각 계산해 작은 값을 남긴다. |
| 11 | 문자열 압축 | 연속해서 반복되는 문자를 문자+반복횟수 형태로 압축한다. 반복횟수 1은 생략한다. | 마지막에 센티넬 문자를 붙이고 이전 문자와 같은 동안 카운트한다. 문자가 바뀌면 문자와 카운트를 결과에 반영한다. |
| 12 | 암호 | `#`과 `*` 7개로 표현된 이진 신호를 대문자 문자열로 해석한다. | 7글자씩 자르고 `# -> 1`, `* -> 0`으로 바꾼 뒤 2진수를 10진수로 변환해 ASCII 문자로 캐스팅한다. |

## 섹션 2. Array

| 번호 | 문제 | 간략 설명 | 풀이 핵심 |
| --- | --- | --- | --- |
| 1 | 큰 수 출력하기 | 수열에서 바로 앞 수보다 큰 수를 출력한다. 첫 번째 수는 항상 출력한다. | 첫 원소를 먼저 넣고, `arr[i] > arr[i - 1]`인 원소만 결과에 추가한다. |
| 2 | 보이는 학생 | 앞에서 봤을 때 이전 모든 학생보다 키가 큰 학생 수를 센다. | 현재까지의 최대 키를 유지하고, 더 큰 키가 나오면 카운트와 최대값을 갱신한다. |
| 3 | 가위바위보 | A와 B가 낸 가위바위보 결과를 라운드별로 출력한다. | 같은 값이면 `D`, A가 이기는 조합 `(1,3), (2,1), (3,2)`이면 `A`, 나머지는 `B`로 처리한다. |
| 4 | 피보나치 수열 | N항까지의 피보나치 수열을 출력한다. | `arr[0]=1`, `arr[1]=1`에서 시작해 `arr[i]=arr[i-1]+arr[i-2]`로 채운다. 배열 없이 직전 두 값만으로도 가능하다. |
| 5 | 소수(에라토스테네스 체) | 1부터 N까지의 소수 개수를 센다. | boolean 배열을 두고 2부터 아직 지워지지 않은 수를 소수로 카운트한 뒤 그 배수들을 지운다. |
| 6 | 뒤집은 소수 | 각 수를 뒤집은 값이 소수이면 출력한다. | 숫자를 `res = res * 10 + n % 10`으로 뒤집고, 2부터 제곱근까지 나누어 소수 여부를 검사한다. |
| 7 | 점수계산 | 연속 정답이면 가산점이 누적되는 방식으로 총점을 계산한다. | 정답이면 연속 카운트를 1 증가시켜 점수에 더하고, 오답이면 연속 카운트를 0으로 초기화한다. |
| 8 | 등수구하기 | 각 학생의 점수 등수를 구한다. 높은 점수가 많을수록 등수가 밀린다. | 각 점수마다 전체 점수를 다시 순회하며 자신보다 큰 점수 개수를 세고 `1 + 개수`를 등수로 둔다. |
| 9 | 격자판 최대합 | N x N 격자에서 각 행, 각 열, 두 대각선 합 중 최댓값을 구한다. | 한 반복문 안에서 행합과 열합을 계산하고, 별도로 두 대각선 합을 계산해 최댓값을 갱신한다. |
| 10 | 봉우리 | 격자에서 상하좌우보다 큰 지점의 개수를 센다. 바깥은 0으로 본다. | 네 방향 배열을 두고 각 칸마다 범위 밖은 낮은 값으로 처리한다. 네 방향보다 모두 크면 봉우리다. |
| 11 | 임시반장정하기 | 1~5학년 동안 같은 반이었던 학생 수가 가장 많은 학생을 찾는다. | 학생 쌍을 비교하며 5개 학년 중 한 번이라도 같은 반이면 카운트하고, 최대 카운트 학생 번호를 답으로 한다. |
| 12 | 멘토링 | 모든 시험에서 멘토가 멘티보다 등수가 앞서는 학생 쌍 개수를 구한다. | 가능한 모든 `(멘토, 멘티)` 쌍에 대해 각 시험의 등수 위치를 찾고, 모든 시험에서 멘토 위치가 더 앞이면 카운트한다. |

## 섹션 3. Two Pointers, Sliding Window

| 번호 | 문제 | 간략 설명 | 풀이 핵심 |
| --- | --- | --- | --- |
| 1 | 두 배열 합치기 | 정렬된 두 배열을 하나의 정렬 배열로 합친다. | 두 포인터를 두고 작은 값을 결과에 넣는다. 한쪽이 끝나면 남은 원소를 그대로 붙인다. |
| 2 | 공통원소 구하기 | 두 배열의 공통 원소를 오름차순으로 출력한다. | 두 배열을 정렬한 뒤 두 포인터로 비교한다. 같으면 출력하고 둘 다 이동, 작은 쪽만 이동한다. |
| 3 | 최대 매출 | 연속 K일 매출 합의 최댓값을 구한다. | 처음 K개 합을 구한 뒤, 오른쪽 값을 더하고 왼쪽 값을 빼면서 창을 한 칸씩 이동한다. |
| 4 | 연속부분수열 | 연속된 부분수열의 합이 M인 경우의 수를 센다. | `rt`를 늘리며 합을 키우고, 합이 M 이상이면 `lt`를 이동하며 줄인다. 합이 M일 때 카운트한다. |
| 5 | 연속된 자연수의 합 | N을 2개 이상의 연속 자연수 합으로 표현하는 경우를 센다. | 투포인터로 1부터 연속합을 관리하거나, `N`에서 `1+...+k`를 빼고 남은 값이 `k`로 나누어지는지 보는 수학 풀이를 쓴다. |
| 6 | 최대 길이 연속부분수열 | 0을 최대 K번 1로 바꿀 때 만들 수 있는 가장 긴 1의 연속 길이를 구한다. | 오른쪽 포인터를 늘리며 0 개수를 세고, 0 개수가 K를 넘으면 왼쪽 포인터를 0 하나가 빠질 때까지 이동한다. |

## 섹션 4. HashMap, HashSet, TreeSet

| 번호 | 문제 | 간략 설명 | 풀이 핵심 |
| --- | --- | --- | --- |
| 1 | 학급 회장 | 후보 문자별 득표 수를 세고 최다 득표자를 찾는다. | `HashMap<Character, Integer>`로 빈도를 세며 현재 최대 득표자를 갱신한다. |
| 2 | 아나그램 | 두 문자열이 같은 문자 구성인지 판별한다. | 첫 문자열의 문자 빈도를 더하고 두 번째 문자열을 보며 빼거나, 두 빈도 맵을 비교한다. 음수가 되거나 없는 문자가 나오면 실패다. |
| 3 | 매출액의 종류 | 길이 K의 연속 구간마다 서로 다른 매출액 종류 수를 출력한다. | 슬라이딩 윈도우와 `HashMap`을 같이 쓴다. 새 값을 더하고 빠지는 값은 빈도를 줄여 0이면 삭제한다. |
| 4 | 모든 아나그램 찾기 | 긴 문자열에서 짧은 문자열과 아나그램인 부분 문자열 개수를 센다. | 목표 문자열 빈도 맵을 만들고, 길이만큼의 윈도우 빈도 맵을 슬라이딩하며 두 맵이 같은지 비교한다. |
| 5 | K번째 큰 수 | 카드 3장을 뽑아 만들 수 있는 합 중 K번째로 큰 수를 구한다. | 3중 반복으로 모든 합을 만들고 `TreeSet`을 내림차순으로 유지한다. K번째 값을 꺼내고 없으면 `-1`. |

## 섹션 5. Stack, Queue

| 번호 | 문제 | 간략 설명 | 풀이 핵심 |
| --- | --- | --- | --- |
| 1 | 올바른 괄호 | 괄호 문자열이 올바르게 짝지어졌는지 판별한다. | `(`는 스택에 넣고 `)`는 하나 꺼낸다. 꺼낼 것이 없거나 마지막에 스택이 남으면 `NO`. |
| 2 | 괄호문자제거 | 괄호 안의 문자를 제거하고 괄호 밖 문자만 남긴다. | 문자를 순회하며 스택에 넣고, `)`를 만나면 `(`가 나올 때까지 제거한다. 마지막 스택의 문자를 순서대로 출력한다. |
| 3 | 크레인 인형뽑기 | 보드에서 인형을 뽑아 바구니에 쌓고 같은 인형이 연속되면 터뜨린다. | 각 move마다 위에서부터 0이 아닌 값을 찾는다. 바구니 스택의 top과 같으면 pop하고 터진 개수에 2를 더한다. |
| 4 | 후위식 연산 | 후위표기식 수식을 계산한다. | 숫자는 스택에 push, 연산자는 두 값을 pop해서 `lt op rt` 순서로 계산 후 push한다. |
| 5 | 쇠막대기 | 괄호 표현에서 잘린 쇠막대기 조각 수를 센다. | `(`는 스택 push. `)`에서 직전 문자가 `(`이면 레이저이므로 pop 후 현재 스택 크기만큼 더하고, 아니면 막대 끝이므로 pop 후 1을 더한다. |
| 6 | 공주구하기 | 원형으로 앉은 사람 중 K번째마다 제거할 때 마지막 남는 번호를 구한다. | 큐에 1..N을 넣고 K-1번은 앞에서 빼 뒤로 보내며, K번째는 제거한다. 하나 남을 때까지 반복한다. |
| 7 | 교육과정설계 | 필수 과목 순서가 수업 계획 안에서 지켜지는지 확인한다. | 필수 과목을 큐에 넣고 계획을 순회한다. 필수 과목을 만나면 큐 앞과 같아야 하며, 끝에 큐가 비어야 `YES`. |
| 8 | 응급실 | 위험도에 따라 진료 순서가 정해질 때 특정 환자가 몇 번째로 진료받는지 구한다. | `(위치, 위험도)`를 큐에 넣고, 현재 환자보다 위험도 높은 환자가 큐에 있으면 뒤로 보낸다. 진료될 때마다 순서를 증가시키고 목표 위치면 반환한다. |

## 섹션 6. Sorting and Searching

| 번호 | 문제 | 간략 설명 | 풀이 핵심 |
| --- | --- | --- | --- |
| 1 | 선택정렬 | 배열을 선택정렬로 오름차순 정렬한다. | 현재 위치부터 끝까지 최솟값 인덱스를 찾고 현재 위치와 교환한다. |
| 2 | 버블정렬 | 배열을 버블정렬로 오름차순 정렬한다. | 인접한 두 값을 비교해 큰 값을 오른쪽으로 보내는 과정을 끝 범위를 줄이며 반복한다. |
| 3 | 삽입정렬 | 배열을 삽입정렬로 오름차순 정렬한다. | 두 번째 원소부터 key로 잡고, 왼쪽의 큰 값들을 한 칸씩 밀어 key가 들어갈 위치에 넣는다. |
| 4 | LRU | 캐시 크기와 작업 순서가 주어질 때 최종 캐시 상태를 구한다. | 작업이 캐시에 있으면 해당 값을 앞으로 이동한다. 없으면 뒤에서부터 한 칸씩 밀고 맨 앞에 넣는다. |
| 5 | 중복확인 | 수열에 중복 값이 있는지 판별한다. | 정렬 후 인접 원소가 같은지 확인하거나 `Set`에 넣으며 이미 있는 값인지 확인한다. |
| 6 | 장난꾸러기 | 정렬된 줄에서 키 순서를 바꾼 두 학생의 위치를 찾는다. | 원본 배열과 정렬된 배열을 비교해 값이 다른 인덱스 2개를 출력한다. |
| 7 | 좌표 정렬 | 여러 좌표를 x 오름차순, x가 같으면 y 오름차순으로 정렬한다. | 좌표 객체 또는 배열을 만들고 comparator에서 x를 먼저 비교한 뒤 y를 비교한다. |
| 8 | 이분검색 | 정렬된 배열에서 특정 값의 위치를 찾는다. | 배열을 정렬하고 `lt`, `rt`, `mid`로 탐색 범위를 절반씩 줄인다. 찾으면 1-based 위치를 출력한다. |
| 9 | 뮤직비디오 | N곡을 M개의 DVD에 순서대로 담을 때 필요한 최소 DVD 용량을 구한다. | 결정 알고리즘. 용량 후보를 이분검색하고, 해당 용량으로 필요한 DVD 개수를 세어 `<= M`이면 더 작은 용량을 시도한다. |
| 10 | 마구간 정하기 | C마리 말을 마구간에 배치할 때 가장 가까운 두 말의 최대 거리를 구한다. | 좌표를 정렬하고 거리 후보를 이분검색한다. 해당 거리 이상으로 말을 몇 마리 배치할 수 있는지 세어 가능하면 거리를 키운다. |

## 섹션 7. Recursive, Tree, Graph

| 번호 | 문제 | 간략 설명 | 풀이 핵심 |
| --- | --- | --- | --- |
| 1 | 재귀함수 | 1부터 N까지 재귀로 출력한다. | `DFS(n - 1)`을 먼저 호출하고 그 뒤 `n`을 출력하면 오름차순이 된다. 종료 조건은 `n == 0`. |
| 2 | 이진수 출력 | 10진수 N을 재귀로 2진수 형태로 출력한다. | `DFS(n / 2)`를 먼저 호출한 뒤 `n % 2`를 출력한다. 종료 조건은 `n == 0`. |
| 3 | 팩토리얼 | N! 값을 재귀로 계산한다. | `n == 1`이면 1, 아니면 `n * DFS(n - 1)`을 반환한다. |
| 4 | 피보나치 재귀 | 재귀와 메모이제이션으로 피보나치 값을 구한다. | 단순 재귀는 중복 호출이 많으므로 배열에 계산 결과를 저장한다. `fib[n]`이 있으면 바로 반환한다. |
| 5 | 이진트리순회 | 이진트리를 전위, 중위, 후위 순회한다. | 현재 노드 처리 위치에 따라 전위(부모-왼쪽-오른쪽), 중위(왼쪽-부모-오른쪽), 후위(왼쪽-오른쪽-부모)가 된다. |
| 6 | 부분집합 구하기 | 1..N 원소로 만들 수 있는 부분집합을 출력한다. | 각 숫자마다 포함/미포함 두 갈래로 DFS를 탄다. 레벨이 N+1이 되면 체크된 원소를 출력한다. |
| 7 | 이진트리 레벨탐색 | 이진트리를 BFS로 레벨 순서대로 출력한다. | 큐에 루트 노드를 넣고, poll한 노드의 왼쪽/오른쪽 자식을 큐에 넣는다. 큐가 빌 때까지 반복한다. |
| 8 | 송아지 찾기 1 | 현재 위치 S에서 목표 위치 E까지 `+1`, `-1`, `+5` 이동으로 가는 최소 횟수를 구한다. | BFS로 위치를 탐색한다. 방문 배열로 중복 방문을 막고, 목표 위치가 나오면 현재 레벨이 최단 거리다. |
| 9 | 말단노드까지의 가장 짧은 경로 DFS | 이진트리 루트에서 가장 가까운 말단 노드까지의 간선 수를 DFS로 구한다. | 자식이 모두 없는 노드가 말단이다. 왼쪽/오른쪽 경로 중 작은 값을 반환하되, 한쪽 자식만 있는 경우 처리를 주의한다. |
| 10 | 말단노드까지의 가장 짧은 경로 BFS | 이진트리 루트에서 가장 가까운 말단 노드까지의 간선 수를 BFS로 구한다. | 레벨 순서로 탐색하다가 처음 만나는 말단 노드의 레벨을 반환한다. 최단 거리는 BFS가 더 자연스럽다. |
| 11 | 그래프와 인접행렬 | 무방향, 방향, 가중치 방향 그래프를 배열로 표현하는 방법을 익힌다. | 정점 번호를 인덱스로 쓰는 2차원 배열을 만든다. 무방향은 `graph[a][b]`와 `graph[b][a]`를 모두 표시하고, 방향은 `graph[a][b]`만 표시한다. 가중치는 1 대신 비용을 저장한다. |
| 12 | 경로탐색 DFS | 방향 그래프에서 1번 정점에서 N번 정점까지 가는 모든 경로 수를 센다. | 인접행렬과 방문 배열을 사용한다. 현재 정점이 N이면 카운트하고, 방문하지 않은 다음 정점으로 DFS를 진행한다. |
| 13 | 경로탐색 인접리스트 | 같은 경로탐색 문제를 인접리스트로 푼다. | 간선 수가 적을 때 인접리스트가 효율적이다. 현재 정점의 리스트만 순회하며 방문 체크 후 DFS, 복귀 시 체크 해제한다. |
| 14 | 그래프 최단거리 BFS | 1번 정점에서 각 정점까지의 최단 거리를 구한다. | 인접리스트와 큐를 사용한다. 시작 정점 거리를 0으로 두고, 방문하지 않은 인접 정점의 거리를 `현재 거리 + 1`로 기록한다. |

## 풀이 패턴 빠른 선택

| 상황 | 먼저 떠올릴 패턴 |
| --- | --- |
| 문자열을 한 번 훑으며 개수, 변환, 필터링을 한다 | 단순 순회, `StringBuilder`, 문자 API |
| 양끝에서 조건을 맞추며 교환하거나 검사한다 | 투포인터 |
| 연속된 구간의 합, 길이, 종류 수를 다룬다 | 슬라이딩 윈도우 |
| 빈도, 존재 여부, 중복 여부가 중요하다 | `HashMap`, `HashSet` |
| 가장 큰 값 순서, 정렬된 중복 제거가 필요하다 | `TreeSet`, 정렬 |
| 괄호, 되돌리기, 직전 값 비교가 중요하다 | 스택 |
| 순서대로 처리하되 뒤로 보내거나 레벨 탐색한다 | 큐, BFS |
| "최소 용량의 최댓값", "최대 거리의 최솟값"처럼 답을 정해 검증한다 | 결정 알고리즘 + 이분검색 |
| 모든 선택 조합 또는 모든 경로를 탐색한다 | DFS, 백트래킹 |
| 최단 횟수나 최단 간선 거리를 구한다 | BFS |
